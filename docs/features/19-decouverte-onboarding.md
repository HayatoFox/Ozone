# Fonctionnalités — Découverte & onboarding

Réf. : [02-serveurs](02-serveurs.md) · [10-roles-permissions](10-roles-permissions.md).

## Onboarding (nouveaux membres)
- [ ] **Questions d'onboarding** : à l'arrivée, le membre choisit des centres d'intérêt → débloque des **salons** et **rôles** correspondants.
- [ ] **Salons par défaut** présentés en premier, salons recommandés.
- [ ] Modes : **standard** vs **avancé** (personnalisation des questions, prérequis).
- [ ] État `pending` tant que l'onboarding n'est pas terminé (accès restreint).
- [ ] Prévisualisation de l'onboarding côté admin, exigences (salon de règles, etc.).

## Écran de bienvenue (welcome screen)
- [ ] Message de bienvenue, description du serveur, **salons mis en avant** (avec emoji + description) menant directement au bon endroit.
- [ ] Affiché à la première visite, configurable par l'admin.

## Règles & passage de vérification
- [ ] **Salon de règles**, écran d'acceptation des règles avant participation (membership screening).
- [ ] Niveaux de vérification (voir [11-moderation](11-moderation-securite.md)).

## Découverte de serveurs
- [ ] **Annuaire de découverte** : recherche par mot-clé, **catégories**, tri (membres, activité).
- [ ] Fiche serveur : description, nombre de membres en ligne/total, langue, aperçu, bouton rejoindre.
- [ ] Critères d'éligibilité (taille, modération, règles), **métadonnées de découverte** (mots-clés, catégories) côté admin.
- [ ] **Serveurs en vedette / recommandés**, serveurs similaires.
- [ ] **Répertoires (hubs)** : pages listant des serveurs (ex. écoles, communautés).

## Definition of Done
- Un admin active la Communauté, configure un écran de bienvenue et un onboarding par centres d'intérêt débloquant rôles/salons, publie le serveur en découverte avec catégories ; un nouvel arrivant le trouve via recherche, accepte les règles, choisit ses intérêts et atterrit dans les bons salons.
