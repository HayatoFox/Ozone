# Fonctionnalités — Amis & relations

Réf. : [03-modele-de-donnees](../03-modele-de-donnees.md#relationship-amisblocages) · [07-messages-prives](07-messages-prives.md).

## Amis
- [ ] **Envoyer une demande d'ami** par pseudo unique.
- [ ] **Accepter / refuser / annuler** une demande.
- [ ] Liste d'amis avec onglets : **En ligne**, **Tous**, **En attente** (entrantes/sortantes), **Bloqués**, **Ajouter un ami**.
- [ ] Statut & activité de chaque ami en direct (présence), tri/recherche.
- [ ] **Surnom d'ami** (alias privé visible seulement par soi).
- [ ] Retirer un ami.
- [ ] Actions rapides : message, appel vocal/vidéo, voir le profil, inviter sur un serveur.

## Paramètres de demandes d'ami
- [ ] Qui peut m'ajouter : **tout le monde** / **amis d'amis** / **membres de serveurs communs** (combinables).

## Blocage
- [ ] **Bloquer** un utilisateur : masque ses messages, empêche MP/appels/demandes, le retire des amis.
- [ ] Liste des bloqués, débloquer.
- [ ] « Ignorer » (option plus douce que bloquer : masque sans notifier — si retenu).

## Notes
- [ ] **Note privée** libre sur n'importe quel utilisateur (visible seulement par soi).

## Présence sociale
- [ ] Voir **serveurs en commun** et **amis en commun** sur un profil.
- [ ] Indicateur « joue à / écoute » sur la liste d'amis.
- [ ] Notifications d'amis qui se connectent (option).

## Definition of Done
- Un utilisateur envoie une demande d'ami par pseudo, l'autre accepte, ils voient leur présence mutuelle et serveurs communs, l'un attribue un surnom privé à l'autre, puis teste le blocage (les messages disparaissent, les MP sont bloqués) et le déblocage.
